> Перед настройкой статичесого IP адреса познакомимся с сетевыми настройками и утилитами в Debian.

## Введение: Зачем нужны разные системы?
В Debian (и Linux в целом) существует несколько способов настройки сети. Это связано с разными сценариями использования:

- Серверы требуют простоты, стабильности и предсказуемости

- Десктопы/ноутбуки нуждаются в гибкости (переключение между Wi-Fi, VPN, мобильным интернетом)

Debian не навязывает одно решение, а позволяет выбрать подходящее под ваши задачи . Ниже разобраны основные инструменты.

1. ifupdown (классический) — наш случай
Что это: Традиционная система настройки сети в Debian, которая существует много лет. Управляется через файл /etc/network/interfaces.

Как работает: При загрузке системы или при выполнении команд ifup/ifdown, система читает конфигурационный файл и применяет настройки для каждого интерфейса .

Для чего используется:

- Идеален для серверов и минимальных установок

- Подходит для статичных конфигураций (один провод, постоянное место)

- Не требует дополнительных сервисов и зависимостей

Почему в нашей системе он: При установке Debian 13 без графического окружения и с минимальным набором пакетов, ifupdown устанавливается по умолчанию, так как это самый легковесный и надежный вариант для сервера .

Команды управления:

`sudo systemctl restart networking  # перезапуск всей сети`

`sudo ifup enp1s0                   # поднять интерфейс`

`sudo ifdown enp1s0                 # опустить интерфейс`

Где живет конфигурация: /etc/network/interfaces и файлы в /etc/network/interfaces.d/

2. NetworkManager (современный для десктопов)
Что это: Современный менеджер сети, который автоматически управляет подключениями. Разработан специально для настольных систем и ноутбуков .

Как работает: Запускается как фоновый сервис (systemd), отслеживает изменения окружения (подключение кабеля, появление Wi-Fi сетей, переключение VPN) и автоматически переключает профили .

Для чего используется:

Десктопы и ноутбуки с графической средой (GNOME, KDE, XFCE)

Сценарии с частым переключением между сетями (офис/дом/Wi-Fi/мобильный интернет)

Настройка Wi-Fi, VPN, мобильных широкополосных соединений

Почему его НЕТ в нашей системе: NetworkManager тянет за собой множество зависимостей (включая графические библиотеки), что противоречит философии минимального сервера.

Инструменты управления:

- nmcli — мощная командная строка 

- nmtui — текстовый псевдографический интерфейс (удобен для начинающих) 

- графические апплеты в GNOME/KDE

Пример настройки статического IP через nmcli:

`nmcli con mod "Wired connection 1" ipv4.addresses 192.168.1.100/24`

`nmcli con mod "Wired connection 1" ipv4.gateway 192.168.1.1`

`nmcli con mod "Wired connection 1" ipv4.method manual`

`nmcli con up "Wired connection 1"`

3. connman (альтернативный легковесный менеджер)
Что это: Connection Manager — легковесный менеджер сети, разработанный Intel. Создан как более простая и быстрая альтернатива NetworkManager .

Как работает: Модульная система с плагинами для DHCP, DNS-кэширования и т.д. Очень быстро реагирует на изменения сети (мгновенно настраивается при подключении кабеля).

Для чего используется:

- Встраиваемые системы, IoT-устройства

- Мобильные платформы (например, Sailfish OS)

- Ситуации, где NetworkManager слишком тяжел, но нужна автоматизация

Почему может быть проблемой: Если connman установлен в системе и активен, он будет конфликтовать с ifupdown, так как оба попытаются управлять сетевыми интерфейсами. Его нужно либо удалить, либо остановить.

Как отключить, если обнаружен:

`sudo systemctl disable connman --now`

`sudo apt remove connman   # полное удаление`

4. systemd-networkd (современный для серверов)
Что это: Компонент systemd, который управляет сетью. Поставляется вместе с systemd (который в Debian 13 уже есть) .

Как работает: Через файлы .network в /etc/systemd/network/. Интегрирован с systemd и может работать в связке с systemd-resolved для DNS.

Для чего используется:

- Современные серверы (альтернатива ifupdown)

- Контейнеры и виртуализация

- Сложные конфигурации (bonding, bridging, VLAN) 

Почему НЕ используется по умолчанию в минимальной установке: В Debian 13 systemd-networkd не включен по умолчанию в минимальных установках, хотя сам systemd установлен. Его нужно явно активировать . Некоторые администраторы предпочитают его за более современный подход и интеграцию.

Как включить (если нужно):

`sudo systemctl enable --now systemd-networkd`

`sudo systemctl enable --now systemd-resolved  # для DNS`

Пример конфигурации (/etc/systemd/network/20-wired.network):
```
ini
[Match]
Name=enp1s0

[Network]
Address=192.168.1.100/24
Gateway=192.168.1.1
DNS=8.8.8.8
```
5. netplan (надстройка от Ubuntu)
Что это: Инструмент от Ubuntu, который работает как абстракция поверх systemd-networkd или NetworkManager.

Как работает: Вы пишете конфигурацию в YAML-файле, а netplan apply преобразует её в настройки для нижележащего менеджера.

Для чего используется: В основном в Ubuntu и её производных. В чистом Debian встречается редко, но может быть установлен.

Пример: /etc/netplan/01-netcfg.yaml
```
yaml
network:
  version: 2
  renderer: networkd
  ethernets:
    enp1s0:
      addresses:
        - 192.168.1.100/24
      gateway4: 192.168.1.1
```
Как проверить, что активно в вашей системе
Вот таблица для быстрой диагностики :

Команда проверки	Если активен →	Что это значит
`systemctl status networking`	active	Используется ifupdown

`systemctl status NetworkManager` 	active	Используется NetworkManager

`systemctl status systemd-networkd` 	active	Используется systemd-networkd

`systemctl status connman`	active	Используется connman

`ls /etc/netplan/*.yaml`	файлы есть	Возможно, используется netplan

> Важно: Активным должен быть ТОЛЬКО ОДИН менеджер для каждого интерфейса. Если запущены несколько, они будут конфликтовать .

Почему в минимальной установке Debian 13 используется ifupdown

- Минимализм: ifupdown не требует дополнительных зависимостей

- Простота: Один файл конфигурации с понятным синтаксисом

- Стабильность: Работает десятилетиями без изменений

- Предсказуемость: Не делает ничего автоматически "за спиной" администратора
